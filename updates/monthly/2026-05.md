# May 2026 Research Summary

The core story of May is that AI tooling is graduating from "cool capability" to "enterprise infrastructure." Cursor shipped persistent cloud dev environments for agents and security review agents. Microsoft released Agent 365 as a governance layer for enterprise AI. Notion opened a developer platform. The pattern across all of it: the products winning aren't the ones with the best models — they're the ones building the organizational plumbing that makes AI actually deployable at scale.

There's a secondary thread running underneath all this: the PM role itself is being renegotiated. Teresa Torres ran an engineering sprint to ship AI-generated Opportunity Solution Trees. Owen Williams at Stripe vibe coded Protodash into a design review standard. Non-PMs are shipping directly to production with AI tools. Meanwhile, Max Schoening is arguing that agency matters more than skills, and the "vibe coding to agentic engineering" boundary is collapsing in ways that should concern anyone building in that space.

## What Matters

- **The environment becomes the product.** Cursor shipping persistent cloud dev environments means agents can now operate inside configured systems with repos, dependencies, credentials, and build access — not just stateless context. This shifts the bottleneck from "can the AI write code?" to "can the AI actually verify and ship it?" For teams evaluating AI coding tools, ownership of the environment is now the differentiator. Cursor, Windsurf, and VS Code are no longer competing on autocomplete quality; they're competing on who owns the infrastructure agents run inside.

- **Enterprise adoption requires governance, not just capability.** Microsoft shipped Agent 365 (identity, audit, compliance layer for enterprise agents). Cursor added model allowlists, spend caps, and usage analytics to Teams/Enterprise. Notion learned that transparency in agent actions was table stakes for user trust. The pattern is consistent: individual adoption doesn't require these features, but team and enterprise adoption can't happen without them. If you're building agent features, your audit log isn't a compliance tax — it's a product feature that unlocks your entire enterprise market.

- **The PM value proposition is shifting from execution to judgment.** Simon Willison warned that vibe coding tools and agentic systems are converging without clear safety boundaries. Max Schoening argued that skills are increasingly offloadable to AI, but agency — the disposition to own outcomes and make decisions under uncertainty — is durable. Harrison Chase flagged that closed model costs are unsustainable for agent workloads at scale, forcing architectural choices now that determine cost structure later. PMs who treat "building with AI" as solving the judgment problem (what should we build?) rather than the execution problem (how fast can we ship?) are positioning themselves to survive the next two years.

## Essential Resources (3)

If your eng team is moving toward agentic workflows, this is the actual pattern to study.

2. **GitHub Blog** — [Validating Agentic Behavior When "Correct" Isn't Deterministic](https://github.blog/ai-and-ml/generative-ai/validating-agentic-behavior-when-correct-isnt-deterministic/) — The framework for thinking about agent correctness when there's no single right answer. Behavioral validation, invariant checking, and trace-level evaluation are how you actually ship agents to production without breaking things.

3. **Notion** — [What We Learned During the Custom Agents Beta](https://www.notion.com/blog/what-we-learned-during-the-custom-agents-beta) — First-party post-mortem on shipping autonomous agents inside a real productivity tool. Autonomy as a dial (not a switch), transparency as table stakes, and context as moat — all directly applicable to any agentic feature you're building.
