# May 2026 Research Summary

May was the month when AI infrastructure became unmistakably real — not because models got smarter, but because the operational layer finally caught up. Cursor shipped persistent cloud development environments for agents, Microsoft launched Agent 365 as a governance control plane, and Notion pushed agents directly into Slack channels. The signal is clear: we've moved past "can AI do this?" to "can enterprises actually deploy this safely, auditably, and at scale?"

The secondary theme running through everything this month is that the PM role is being renegotiated in real time. Teresa Torres ran an engineering sprint to build AI-generated Opportunity Solution Trees. Stripe's Owen Williams vibe-coded Protodash and transformed design reviews. Non-PMs are shipping directly to production using Cursor and Claude Code. The question teams are actively grappling with now isn't whether PMs should learn to code — it's whether the traditional product workflow even makes sense when a developer can ship a feature overnight.

## What Matters

- **The environment is the moat for coding agents.** Cursor's persistent cloud dev environments fundamentally changed what agents can accomplish. Before: agents could write code suggestions. After: agents can clone repos, install dependencies, run builds, and submit PRs autonomously. This isn't an incremental improvement — it's the infrastructure required for agents to actually close tickets instead of just drafting solutions. The ROI math flips when an agent can own a task end-to-end without human scaffolding. Every team piloting agentic workflows needs to ask: which coding tools actually provide this environment context, and which ones force developers to bootstrap agent context manually each time?

- **Enterprise adoption requires governance, not just capability.** Microsoft's Agent 365, Cursor's security agents, and Notion's custom agent controls all shipped in May because enterprises had hit a hard wall: "Who's responsible when an agent does something wrong?" The products winning enterprise conversations aren't the ones with the best models — they're the ones with audit trails, managed identities, approval workflows, and spend controls. This is the checklist: model allowlists, spend caps, usage visibility, and agent action logging. Without these, procurement stalls. With them, you're unblocking a completely different buyer conversation.

- **The PM value proposition is shifting from execution to judgment.** Max Schoening's argument that "agency beats skills" and Teresa Torres's push-back on the "product builder" identity both point at the same reality: as AI makes shipping trivial, the irreplaceable PM contribution becomes the decision about *what* to build, not the ability to *build* it. Notion's spec-driven development workflow assumes PMs write better specs — not because specs are fun, but because the quality of the spec is now the bottleneck. Teams building with AI are discovering that more software faster is actually a liability if the software solves the wrong problem. The PMs who stay relevant are the ones who can say no as confidently as they can say yes.

## Essential Resources (3)

1. **Lenny Rachitsky** — [Why SaaS Freemium Playbooks Don't Work in AI](https://www.lennysnewsletter.com/p/why-saas-freemium-playbooks-dont) — AI products have real per-user inference costs, so traditional freemium math breaks; the winning pattern is capability-gating (limiting what free users can do) rather than usage-gating.

2. **GitHub Engineering Blog** — [Validating Agentic Behavior When "Correct" Isn't Deterministic](https://github.blog/ai-and-ml/generative-ai/validating-agentic-behavior-when-correct-isnt-deterministic/) — Stop testing whether agents produce the "right" output and start testing whether they stay within acceptable constraints; behavioral validation is the framework that actually works for non-deterministic systems.

3. **Notion (Akshay Kothari)** — [What We Learned During the Custom Agents Beta](https://www.notion.com/blog/what-we-learned-during-the-custom-agents-beta) — Users don't trust agents they can't audit (even correct ones), autonomy needs to be a dial not a switch, and repetitive workflows (not creative work) drive retention — first-party insights from a company shipping agents at scale.
