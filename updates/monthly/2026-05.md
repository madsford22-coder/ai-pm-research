# May 2026 Research Summary

The big story this month is that agentic AI crossed from "interesting capability" to "operational infrastructure." We're not debating whether agents work anymore — we're debating how to govern them, price them, and integrate them into existing workflows without breaking trust. The week Anthropic hit $47B run-rate revenue, LangSmith shipped an agent that debugs other agents, and Cursor built security review into the core product tells you everything about where the market thinks this is heading.

The real tension underneath everything is shifting from capability to control. As agents become the default execution layer — handling code, content, payments, data — the PM work moves upstream: who decides what the agent can do, what oversight looks like, and what happens when it fails. This month showed three distinct answers: Notion treats transparency as table stakes (users need to audit what agents did), Microsoft built governance into Agent 365 (compliance teams need visibility), and Cursor gated security features to Teams plans (enterprises need control). These aren't competing visions. They're layers of the same answer.

## What Matters

- **Agents are now infrastructure, not features — which means governance is the real product.** Cursor shipped always-on security agents only to Teams/Enterprise. Microsoft launched Agent 365 as a control plane for agent identity and compliance. Notion learned that users won't trust agents they can't audit, even correct ones. The pattern is consistent: teams shipping agentic capabilities at scale are discovering that "can the agent do this?" is less important than "can I see what it did and control what happens next?" For PMs, this means your audit logs and approval workflows aren't backend concerns — they're product features that determine enterprise adoptability.

- **The cost structure of agentic work broke the SaaS freemium playbook.** Lenny's deep dive showed why traditional SaaS freemium (unlimited free tier, convert on premium features) fails when inference costs are real. Vercel switched to per-unit billing for functions. Dan Shipper's report showed Every's headcount grew 7.5x despite automation because cheap execution drives demand for better direction. The lesson: don't price AI features like software. Price them like you're paying for compute — because you are. Capability-limited free tiers (users can ask Claude 3 times/day, or they can ask Sonnet unlimited) protect margins while letting users feel real value.

- **The interface is no longer the product — context is.** OpenAI added financial accounts to ChatGPT. Notion pushed agents into private Slack channels. Figma Make connected directly to local codebases. The pattern is that products winning agentic adoption are the ones that integrate into existing context, not the ones that create new surfaces. For PMs, this means your AI feature lives where the work already happens (Slack, Jira, GitHub), not in a tab users have to remember to visit. The teams that figure this out first are the ones building moats that competitors can't easily copy.
