# May 2026 Research Summary

The line between "vibe coding" and "agentic engineering" is collapsing faster than anyone expected — and most teams building in that space haven't noticed the safety gap yet. What started as casual AI-assisted prototyping is quietly turning into autonomous systems that send emails, write to databases, and take actions without explicit human approval at each step. Meanwhile, the infrastructure layer for running agents at enterprise scale shipped this month, which means the conversation is shifting from "can we do this?" to "can we govern this?"

The secondary thread running through May is about who actually pays for AI. Anthropic's Claude Code A/B test on pricing, OpenAI's shift to ads in ChatGPT, and LangChain's bet on open-source models all point to the same cost problem: the economics of agentic workflows are broken for closed models at scale. Teams are hitting this wall right now, and it's reshaping architecture decisions.

## What Matters

- **Cursor shipped always-on security agents for enterprise, making AI coding tools procurement-safe.** Two weeks after releasing model controls and spend management, Cursor added a Security Reviewer (checks every PR for vulnerabilities) and Vulnerability Scanner (continuous codebase auditing) as gated Teams/Enterprise features. This removes the single biggest blocker enterprise security teams have had: "Does AI-generated code introduce new vulnerabilities?" By packaging oversight into the product itself, Cursor solved the approval problem and created a hard upgrade incentive. This is the playbook for any developer tool moving upmarket — nail individual velocity first, then add governance.

- **Notion's agent beta revealed that transparency trumps accuracy for user trust.** In their post-mortem on Custom Agents, Notion found that users needed to see what the agent *did*, not just whether it produced correct output — black-box agents generated support tickets even when they succeeded. This one finding should reshape how you think about agent UX: audit logs aren't compliance theater, they're the product. Notion also shipped agents in private Slack channels, moving agents into where decisions actually happen rather than where work gets documented.

- **The PM-less shipping problem is real and forcing a reckoning with what PMs actually own.** Lenny's Community Wisdom piece surfaced the tension that's showing up across teams: engineers using Claude Code or Cursor are shipping features directly to production, bypassing traditional PM review. This isn't a process problem — it's a structural one. When building drops from days to hours, the old discovery-definition-delivery loop doesn't fit. Teams are now forced to decide whether PMs own the *decision to ship* or the *strategic context* that informs decisions others make. Those are fundamentally different jobs, and most organizations haven't articulated which one they actually want.
