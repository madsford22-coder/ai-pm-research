# May 2026 Research Summary

May was the month validation stopped being optional. Anthropic's 80x growth number opened the month, but the real signal came from GitHub, Windsurf, and Notion all shipping agent validation infrastructure in the same week — behavioral constraints instead of perfect outputs, code review baked into the editor, transparency logs instead of black boxes. The pattern is unmistakable: as agents move from demos to production, the PM's job shifted from "make it smarter" to "make it auditable."

The enterprise infrastructure layer also crystallized this month. Cursor shipped security agents gated to Teams plans (forcing upgrade conversations), Microsoft flipped Agent 365 to GA (governance becomes table stakes), and even Notion learned the hard way that users won't trust agents they can't see inside. The message to builders is clear: vibe coding gets you to working software, but only governance gets you to production.

## What Matters

- **Agent validation is now a product category, not a design afterthought.** GitHub's post on validating agentic behavior landed the same week as Windsurf's code review-in-editor and Cursor's always-on security agents — all solving the same underlying problem: how do you trust an agent you can't fully predict? The shift from "did the agent produce the right output?" to "did the agent stay within the right constraints?" is how you actually ship agents that don't require a human to second-guess every decision. Notion's Custom Agents beta found that transparency was table stakes for trust — users generated support tickets even for correct outputs because they couldn't see what the agent actually did.

- **Enterprise adoption now requires security and governance to be first-class product features.** Cursor didn't add security review because it makes coding faster — it added it because enterprise procurement won't sign off without it. Microsoft's Agent 365 GA is the control plane for that conversation: identity, audit trails, compliance across every agent interaction. When a product gates a feature to Teams/Enterprise plans (Cursor's security agents, Microsoft's Copilot Cowork connectors), that's signaling where the real upgrade lever is. Individual productivity stopped being the blocker; organizational control is.

- **The cost economics of agents are forcing a reckoning with open-source models.** Harrison Chase named it plainly: "cost of closed models is too high" for the agentic workflows 2026 is building. A single agent run involving research, planning, and execution can cost 50–200x more than a simple completion — and that math breaks for teams running thousands of agents. Sendbird's internal adoption playbook (quests, tokens, leaderboards) works because teams are actually using AI to ship work, not experimenting. The economics only close if you're not paying $0.30 per agent invocation on frontier models.
