# May 2026 Research Summary

May showed a clear inflection: AI tooling stopped being about "what can we build" and started being about "how do we run this safely at scale." OpenAI published their internal Codex safety playbook, Cursor shipped always-on security agents for enterprise, and Microsoft flipped the switch on Agent 365 — a full control plane for managing agents across organizations. The signal isn't new capability announcements; it's governance, audit trails, and the infrastructure that lets enterprises actually say yes.

The other current running through the month is economic pressure. Harrison Chase (LangChain) named the obvious thing nobody was saying publicly: closed model costs are too high for production agent workloads, which means 2026 is when open-source models stop being the scrappy alternative and start being the default for anyone doing multi-step agentic work. That economic shift is already changing architecture decisions.

## What Matters

- **Governance is now table stakes for enterprise AI adoption.** Cursor added model allowlists, spend caps, and continuous security review agents — gated to Teams/Enterprise plans as a forcing function. Microsoft Agent 365 shipped identity, audit trails, and compliance controls that work across any agent. Notion found that users wouldn't trust agents they couldn't audit, even when the agent's output was correct. The pattern is consistent: the tools winning enterprise deals aren't the ones with the most capability; they're the ones that let IT and security teams actually understand what's happening.

- **The "vibe code to production agent" boundary is collapsing and nobody's prepared for it.** Simon Willison flagged that tools designed for casual prototyping (generate an HTML file in 30 seconds, vibe code a quick tool) are accumulating real capabilities — file system access, API calls, persistent state — without users recognizing the jump from "fun experiment" to "this agent can do real damage." Sendbird's marketing team shipped a live e-commerce store in a day with zero engineers. Max Schoening (Notion) countered that more software ≠ better software, and the judgment about what to build remains irreplaceable. The risk and the opportunity sit right next to each other.

- **Token efficiency is the new unit economics problem for agentic products.** GitHub, Cursor, and Harrison Chase all surfaced the same constraint this month: a single agent run involving planning, research, execution, and verification isn't 10 queries' worth of tokens — it's 50–200x more because context grows with each step. Vercel shipped conversation history as table stakes for agents. OpenAI published how they sandbox Codex and gate risky actions. The economics of "let the agent try" break fast when you're running 20-step workflows at scale.

## Essential Resources (3)

1. **OpenAI** — [Running Codex Safely at OpenAI](https://openai.com/index/running-codex-safely) — The clearest published framework for production agent governance: sandboxing, human approval gates, network policies, and agent-native telemetry for audit.

2. **Notion** — [What We Learned During the Custom Agents Beta](https://www.notion.com/blog/what-we-learned-during-the-custom-agents-beta) — First-party post-mortem showing users need visibility into what agents actually do, and that autonomy is a dial you adjust per-task, not a binary switch.

3. **Lenny Rachitsky** — [Why SaaS Freemium Playbooks Don't Work in AI](https://www.lennysnewsletter.com/p/why-saas-freemium-playbooks-dont) — The cost structure argument that explains why capability-limited free tiers outperform usage-limited ones for AI products, with actionable pricing patterns.
