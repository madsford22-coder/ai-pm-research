# May 2026 Research Summary

The line between "AI assists" and "AI executes" disappeared this month. Notion's spec-driven development shows what happens when engineers stop writing code and start writing requirements. OpenAI launched DeployCo, a whole company dedicated to closing the gap between "we bought AI access" and "AI actually changed how we operate." Cursor, Windsurf, and GitHub all shipped features this week that assume agents are doing the work — and humans need guardrails, not just visibility.

The harder signal underneath: enterprise buyers are done with capability theater. They want governance, spend controls, audit trails, and the ability to say "no" to what the AI wants to do. The product work moved from "how do we make the AI smarter?" to "how do we make the AI safe enough to deploy at scale?"

## What Matters

- **The spec is now the bottleneck, not the code.** Notion's Ryan Nystrom walks through a workflow where engineers write detailed specifications, then AI agents implement them. The shift isn't about speed — it's about what PMs actually own. If specs are the new foundation, then the PM who writes better specifications ships faster. The failure mode is underspecification creating agent confusion; the unlock is spec quality becoming the visible constraint instead of hidden in an engineer's judgment.

- **Enterprise adoption unlocked by governance, not capability.** Cursor shipped security review agents for Teams/Enterprise, Microsoft shipped Agent 365 with centralized identity and audit trails, and Notion published what they learned about agent transparency. The pattern is consistent: enterprises don't care about the fanciest agent — they care about knowing what it did and being able to prove it to their security team. DeployCo's existence is OpenAI admitting the same thing: model access isn't the problem; closing the gap from "we have API keys" to "AI changed how we work" is.

- **The cost structure of agentic workflows is breaking closed models.** LangChain's Harrison Chase called it directly: "cost of closed models is too high" for multi-step agents. GitHub published patterns for token efficiency in agentic workflows. The math is brutal — an agent that takes 20 steps compounds costs faster than 20 individual API calls because context grows at each step. Teams building at scale are already hitting the wall; 2026's architectures are being designed around OSS models out of economic necessity, not preference.
