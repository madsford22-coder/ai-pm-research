# April 2026 Research Summary

April is the month when "agents orchestrating agents" stopped being a future scenario and became a shipping priority. Cursor rebuilt its entire interface around parallel agents, OpenAI restructured Codex pricing to lower adoption friction, and Microsoft shipped Council — a feature that runs the same prompt through multiple models simultaneously. The signal is consistent: the interaction model that's winning isn't "one assistant answering one question." It's "multiple agents working in parallel, with humans reviewing and coordinating."

Underneath all of this is a harder problem nobody's solved well yet: what does a human actually do when the agent is doing the work? Simon Willison named the cognitive cost of being a permanent reviewer. Karri Saarinen's six-principle framework for human-agent interaction addresses the gap. And Teresa Torres walked through the "doom loop" — the failure pattern that kills momentum in vibe coding — which turns out to be solved by the same practices PMs are already good at: planning before you build, reviewing as you go, knowing when to stop.

## What Matters

- **Agents are now expected to learn from behavioral signals, not just model updates.** Cursor's Bugbot learns from what developers dismiss vs. accept on pull requests, updating its rules in real time without configuration. Stripe's Veni Singh published real checkout data showing agents are already transacting in production. LangChain's framework distinguishes four layers where agents improve — weight updates, instruction updates, memory updates, and tool updates — and notes that the fastest loops are at the instruction and memory layers, not the model layer. The implication for PMs: the agent that improves fastest in your specific context is not the one with the best base model, it's the one with the best feedback infrastructure.

- **The review burden is the design problem that matters most right now.** Simon Willison articulated what everyone's feeling but hasn't named: using coding agents doesn't eliminate cognitive load, it transforms it — you stop writing and start reviewing, which is a different kind of sustained attention. Karri Saarinen's Linear framework names six principles for human-agent interaction, with "interrupt and correct" and "match autonomy to stakes" being the most underappreciated. Most agent interfaces are designed for happy-path execution; almost none handle mid-task correction gracefully. For any PM shipping agent features, this is where the design work actually is — not in making the agent smarter, but in making the review interface faster and more trustworthy.

- **Pricing and friction are now the primary levers for agentic tool adoption, not capability.** Anthropic grew from $1B to $19B ARR in 14 months partly by deliberately adding onboarding friction (filtering for users who will stick) and using an internal agent called CASH for autonomous growth work. OpenAI moved Codex to pay-as-you-go pricing for Business and Enterprise, removing the upfront commitment friction that was blocking team trials. Wispr Flow eliminated seat minimums and domain restrictions, letting any teammate try for 14 days without credit card or approval. The pattern: the teams winning at agent adoption are optimizing for "easier to trial" and "clearer success metrics," not "more features."

## Essential Resources (3)

1. **Karri Saarinen (Linear CEO)** — [How to Design for Human-Agent Interaction](https://every.to/thesis/how-to-design-for-human-agent-interaction) — Six-principle framework for agent UX built from shipping agent features into Linear's core product, with emphasis on "interrupt and correct" and matching autonomy to stakes.

2. **LangChain** — [Continual Learning for AI Agents](https://blog.langchain.com/continual-learning-for-ai-agents/) — Framework distinguishing four layers where agents improve (weights, instructions, memory, tools) with the practical insight that instruction and memory updates compound faster than model updates.

3. **Stripe (Veni Singh, PM)** — [How Agents, Digital Wallets, and Trust Are Rewriting Checkout](https://stripe.com/blog/product) — Real production data showing agents are already transacting on Stripe's network, with specific findings on trust signals and context-specific conversion patterns.
