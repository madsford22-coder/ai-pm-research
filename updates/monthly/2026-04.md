# April 2026 Research Summary

April was the month the AI product industry stopped arguing about "if" and started shipping "how." Cursor rebuilt its entire interface around parallel agents, Anthropic grew from $1B to $19B ARR in 14 months using internal AI systems for growth, and Stripe's payment data showed agents are already transacting autonomously in production. This wasn't incremental — it was a threshold moment where agentic workflows moved from experiments to operational infrastructure across coding tools, enterprise platforms, and payment systems.

Underneath all the shipping was a consistent tension: as AI handles more execution work, the human role shifts from doing to reviewing — and nobody has designed that experience well yet. From Simon Willison naming the cognitive cost of code review to Karri Saarinen's six principles for human-agent interaction, the pattern was clear. The products winning on adoption aren't the ones that disappear into the background; they're the ones that keep humans visibly in the loop and make the collaboration intentional.

## What Matters

- **The review burden is the new product design problem.** When agents execute work, humans become reviewers — but most product interfaces treat this as an afterthought. Cursor's Canvases turn AI responses into interactive UIs you can actually evaluate. Microsoft's Track Changes in Word let you see exactly what Copilot changed and accept/reject it word-by-word. Linear's agent solves 57% of bugs autonomously, leaving the hardest 43% for humans. The pattern is clear: products that surface agent work legibly — showing what changed, why, and what the trade-offs are — are the ones users actually trust at scale.

- **Context persistence is the infrastructure gap nobody solved yet.** Teresa Torres documented how AI workflows are fragile across devices; LangChain published a framework for how organizations encode tacit knowledge into agents so they actually improve from usage; Cursor's Bugbot shipped learned rules that tune behavior based on team feedback. But the underlying problem remains: most teams haven't built the muscle to capture and apply what their people know. The teams investing in this now — building shared prompt libraries, instrumenting feedback loops, treating agent improvement as a core operational practice — will have a compounding advantage.

- **Autonomy levels are context-dependent, not universal.** Lenny's categorization framework for agent types, Saarinen's "match autonomy to stakes" principle, and Willison's observation that users don't want automation (they want augmentation) all point to the same thing: there's no single setting for how much an agent should do without approval. A bug-fixing agent can run mostly autonomously because errors are visible in tests. A financial transaction agent needs explicit checkpoints because the stakes are real. The products shipping this insight — Microsoft's hybrid agent+workflow automation, GitHub's multi-model "second opinion" feature — are the ones building genuinely usable agentic systems.

## Essential Resources

1. **Linear / Karri Saarinen** — [How to Design for Human-Agent Interaction](https://every.to/thesis/how-to-design-for-human-agent-interaction) — A six-principle framework built from Linear's actual shipping experience: agents work alongside humans (not replacing them), transparency over magic, interrupt and correct mid-task, match autonomy to stakes.

2. **Stripe / Veni Singh** — [How Agents, Digital Wallets, and Trust Are Rewriting Checkout](https://stripe.com/blog/product) — Real transaction data showing agents are already completing purchases autonomously, trust signals move conversion more than friction, and the checkout UX built for humans breaks for agents.

3. **Teresa Torres** — [Vibe Coding Best Practices: Avoid the Doom Loop](https://www.producttalk.org/vibe-coding-best-practices/) — The failure pattern every PM vibe coder hits (skip planning, code gets broken, spiral into patches), why it happens, and what planning discipline actually prevents it.
