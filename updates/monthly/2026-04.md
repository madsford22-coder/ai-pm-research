# April 2026 Research Summary

April made one thing unmistakably clear: the tacit knowledge problem is now the defining constraint for enterprise AI. It's not that agents can't do the work — they can, at scale. It's that organizations haven't figured out how to get what actually matters (the stuff in people's heads, the judgment calls that live in behavior) into the systems that are supposed to improve over time. LangChain published a framework for this, Cursor's Bugbot is learning from it in real time, and Stripe's checkout data shows it's already shaping how customers behave. The question underneath all of it is the same: how do you keep humans in the improvement loop without making it a bottleneck?

The second signal is structural: the product interface itself is the battleground now. Cursor redesigned its entire UI around parallel agents, OpenAI removed friction from Codex pricing, and Microsoft shipped a "Council" feature that runs multiple models at once. These aren't feature releases—they're bets on what the daily workflow actually looks like when agents are doing significant work. The companies that win in the next 18 months won't be the ones with the smartest agents. They'll be the ones with interfaces that make orchestrating many agents feel natural.

## What Matters

- **Tacit knowledge is now the real moat in agent systems.** LangChain's human judgment framework distinguishes between explicit knowledge (documented, codable) and tacit knowledge (lives in how teams actually work). Cursor's Bugbot learns from what developers accept vs. dismiss on pull requests, improving without explicit rules. Stripe's Veni Singh found that agents buying from checkout need explicit trust signals that humans infer automatically. The pattern: every correction an agent receives is either a one-time fix (worthless) or a generalizable update (compounds forever). Most teams only capture the first kind. For PMs: the question isn't "what can the agent do?" It's "how does the agent learn what your users actually care about?"

- **Multi-agent orchestration is becoming the primary interaction model.** Cursor 3.0 redesigned its interface entirely around running parallel agents in different environments (local, cloud, SSH worktrees). GitHub's Copilot CLI shipped `/fleet` for parallel agents. Microsoft's M365 Copilot now runs the same prompt through multiple models simultaneously. Stripe's internal "minions" agents ship 1,300 PRs per week. The convergence is unmissable: the daily workflow is shifting from "chat with one assistant" to "orchestrate many at once." The review burden is the new problem nobody's designed well yet.

- **The review burden is the actual cognitive cost, not the absence of writing.** Simon Willison articulated what most people are feeling: when you delegate to agents, you don't eliminate cognitive load—you transform it. You stop producing and start reviewing, which demands sustained attention. Karri Saarinen's six principles for human-agent interaction (especially "interrupt and correct" and "transparency over magic") show what good design looks like. Teresa Torres's vibe coding doom loop shows what bad design looks like: you skip planning, accumulate broken patches, and end up restarting. The teams getting leverage from agents aren't the ones best at prompting—they're the ones with planning discipline and review practices already in place.

## Essential Resources (3)

1. **Karri Saarinen (Linear)** — [How to Design for Human-Agent Interaction](https://every.to/thesis/how-to-design-for-human-agent-interaction) — Six-principle framework from someone who actually shipped agent features: match autonomy to stakes, enable mid-task correction, preserve user voice, and three others that map directly to what you're probably missing in your roadmap.

2. **LangChain** — [Human Judgment in the Agent Improvement Loop](https://blog.langchain.com/human-judgment-in-the-agent-improvement-loop/) — Framework distinguishing tacit knowledge (hard to extract but invaluable) from explicit knowledge, and how to surface it so agents learn from real work instead of documentation.

3. **Stripe** — [How Agents, Digital Wallets, and Trust Are Rewriting Checkout](https://stripe.com/blog/product) — Data-backed analysis from PM Veni Singh showing agents already transacting in production, which means trust signals now need to be machine-readable and your checkout flows need to work when nobody's reading the UI.
