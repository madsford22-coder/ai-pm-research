# April 2026 Research Summary

Planning discipline, not prompting skill, is turning out to be the actual bottleneck in AI-assisted work. Teresa Torres published a framework for the "doom loop" — the pattern where you skip planning, the AI starts patching patches, and you scrap everything and start over. Meanwhile, Microsoft shipped Council (running multiple models simultaneously to see where they disagree), and OpenAI's $122B raise is explicitly betting on infrastructure capacity, not new model capabilities. The signal across all three is the same: execution is easy now; knowing what to build and reviewing what gets built is where the actual work happens.

This matters because it flips the skill premium. You don't need the best prompting ability — you need the planning and review discipline that PMs already have. That's a genuine advantage in the vibe coding era, and it explains why teams like Stripe can ship 1,300 PRs per week from agents: they're not solving prompting; they're solving specification and review at scale.

## What Matters

- **Planning before prompting stops the doom loop.** Teresa Torres has been vibe coding for six months and documented the exact failure pattern: you skip the spec because the demo looked easy, add scope mid-session, the AI loses context and starts generating broken patches on top of broken patches, and eventually you scrap everything. Her fix is simple — write a one-paragraph spec before you start (what you're building, what done looks like, what you explicitly won't build), keep it visible during the session, and use it to cut off scope creep. This isn't theoretical; it's from someone who's written more software in six months than her entire lifetime prior.

- **Model disagreement is a feature, not a bug.** Microsoft shipped Council to M365 Copilot's Frontier tier, letting you run the same prompt through multiple models simultaneously and see where they agree and diverge. That's philosophically different from what OpenAI and Anthropic are doing — they're trying to build one model you trust completely; Microsoft is building a workflow where you consult multiple models like a council of advisors. For high-stakes decisions (legal review, financial analysis, product strategy), seeing where models disagree is often the most useful signal. The flip side: it may train users to distrust any single model output, which complicates simpler use cases.

- **The coding agent market is large enough for $122B infrastructure bets.** OpenAI's $122 billion funding round explicitly named Codex demand as a driver alongside consumer ChatGPT — a signal that the coding agent market has moved from experimental to infrastructure-scale. The capital is going to compute and global expansion, not new model capabilities. For PMs building on OpenAI APIs, this is a mixed signal: more capacity and lower prices over time, but also OpenAI building increasingly in direct competition with application-layer products. The real decision point: if OpenAI is expanding data residency and regional availability, waiting for them to solve your enterprise compliance problem may be cheaper than solving it yourself.
