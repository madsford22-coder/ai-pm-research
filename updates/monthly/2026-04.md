# April 2026 Research Summary

The default is becoming agentic. Three major signals this month made it explicit: Microsoft shipped Agent Mode as the *default* in Office, OpenAI launched Workspace Agents for teams, and Anthropic's Head of Product described how their entire PM culture is already operating as if agents are the unit of work. This isn't "AI will eventually change workflows" — it's shipping now, with the baseline assumption that the average knowledge worker is an agent user.

The organizational scaling problem is the new bottleneck. Intercom doubled engineering velocity with Claude Code, Linear's agent now solves 57% of bugs autonomously, and Stripe showed agents are already transacting in production. But every team hitting these gains shares the same pattern: they had to build shared infrastructure (prompt libraries, memory systems, evaluation frameworks) and create cultural practices (code review, planning discipline) around the AI tooling. The velocity gains are real; they're just not free.

## What Matters

- **The multi-agent coordination problem is becoming the central product design challenge.** Cursor 3.0 rebuilt its entire interface around parallel agents; Windsurf 2.0 added an Agent Command Center; GitHub shipped `/fleet` for simultaneous agents in Copilot CLI. But as Maggie Appleton diagnosed, agents running in parallel have zero shared context — they'll contradict each other or duplicate work silently. The human becomes the manual integration layer. Lenny's framework for categorizing agents by risk (read-only vs. write-capable, reversible vs. irreversible) is the clearest triage method so far, but the actual UX for managing multi-agent conflicts remains unsolved.

- **Planning discipline is the actual bottleneck in AI-assisted work, not prompting ability.** Teresa Torres documented the "doom loop" — vibe coders skip planning, hit a wall, start patching patches, and eventually restart. Anthropic deliberately added onboarding friction as part of a strategy that took them from $1B to $19B ARR in 14 months. GitHub Copilot's Bugbot learned from user feedback; Linear's agent succeeded because the work was high-volume and well-specified. The pattern across all of it: teams with strong specs, review practices, and evaluation frameworks get velocity multipliers; teams without them get faster shipping in the wrong direction.

- **The cognitive cost of reviewing AI-generated work is the hidden drag on productivity claims.** Simon Willison named something most people feel but haven't articulated: delegating to agents doesn't eliminate mental load, it transforms it. You stop writing and start reviewing, which requires sustained attention across multiple parallel tasks. Karri Saarinen's framework (match autonomy to stakes, enable mid-task corrections, preserve user agency) is the most concrete answer to date, but most agent interfaces are still optimized for happy-path completion, not for the review workflows that actually determine whether the gains stick.
