# April 2026 Research Summary

April made one thing brutally clear: the race for AI-assisted development isn't about better models anymore—it's about better interfaces for managing multiple agents at once. Cursor redesigned its entire editor around parallel agents, OpenAI restructured Codex pricing to remove friction, GitHub shipped `/fleet` for simultaneous multi-agent runs, and Microsoft built Council to let you consult multiple models like advisors. The product battle is happening at the orchestration layer, not the capability layer.

The secondary shift is equally important: planning and review discipline are now the actual bottleneck, not prompting ability. Teresa Torres's doom loop framework, Stripe's 1,300-PR-per-week agent workflow, and Simon Willison's inflection point argument all point to the same thing—teams winning with AI aren't the ones with the best prompts, they're the ones with the strongest specs and the most rigorous review practices. That's a skill set PMs already have.

## What Matters

- **Parallel agents are becoming the primary interaction model.** Cursor 3.0's Agents Window lets you run multiple agents simultaneously across local, cloud, and SSH environments—a ground-up UI redesign that signals this is now the default workflow, not a feature. GitHub's `/fleet` command in Copilot CLI does the same thing for PR generation. The coordination problem is live: how do you review outputs from three agents working simultaneously? How do you spec work for agents vs. humans? These are product questions nobody has fully solved yet.

- **Pricing is shifting from commitment to consumption to enable team adoption.** OpenAI removed upfront commitments from Codex by moving to usage-based billing for Business and Enterprise customers. Wispr Flow collapsed four tiers into one, eliminated seat minimums, and made every invited teammate eligible for a free 14-day trial—a textbook product-led growth move designed to let power users pull teammates in without finance or IT friction. The pattern: agentic tools are variable-usage by nature, so fixed pricing is a growth blocker. Consumption-based pricing gets teams in the door.

- **Planning before prompting is the discipline that separates success from the doom loop.** Teresa Torres, after six months of vibe coding (more software than her entire prior career), identified a specific failure mode: skipping the spec, hitting walls, patching patches until everything is broken. The fix isn't better prompting—it's writing a clear goal before you start, reviewing outputs as you go, and ending sessions when scope creeps. Simon Willison's "inflection point" isn't about capability; it's about the economics of software production becoming structurally different when teams have planning discipline to pair with agent execution.
