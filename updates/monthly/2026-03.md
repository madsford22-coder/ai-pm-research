# March 2026 Research Summary

March confirmed what February hinted at: the era of AI as suggestion is ending, and execution—where agents directly produce commits, PRs, and deployed code—is becoming the default interface. Cursor's self-hosted agents unlocked enterprise procurement, Notion shipped Custom Agents for mainstream teams, and GitHub documented how Squad coordinates multiple agents in parallel. The structural shift isn't about faster models; it's about agents moving from assistant to actor.

Alongside execution came the hard problems: how do you secure agents operating at scale, manage dozens of them across teams, and debug when multi-agent coordination fails? This month showed those aren't unsolved research questions anymore—they're shipped products with reference architectures.

## What Matters

- **Enterprise procurement unlocked by data residency and security**. Cursor shipped self-hosted cloud agents (agents run entirely on your infrastructure, code never leaves your network), Notion expanded data residency to Japan and South Korea, and GitHub documented explicit security architecture for agentic workflows. The common thread: companies with sensitive data were previously locked out of AI tooling. Now the checkbox "does it support our compliance requirements?" is actually checkable. Procurement conversations that were impossible three months ago are now possible.

- **Multi-agent coordination moving from pattern to product**. GitHub's Squad runs multiple agents in parallel inside repositories, coordinating through shared Git state. LangChain rebranded Agent Builder as Fleet—explicitly positioning for enterprise team-wide agent management. Vercel added MCP (Model Context Protocol) support for standardized agent tool discovery. The pattern: single agents are table stakes; the real product value is now in orchestrating multiple agents without them stepping on each other's work or creating coordination bottlenecks.

- **Agent infrastructure converging on managed services instead of DIY**. LangChain shipped Polly (AI debugging agent traces), Sandboxes (secure code execution), Deploy CLI (agent deployment pipelines), and autonomous context compression (preventing agents from exceeding context limits mid-workflow). Vercel published how Notion uses Vercel Sandbox at scale. The signal: teams no longer build context management, observability, sandboxing, or security isolation from scratch—they buy managed versions. This matters because it compresses the time from "we want to deploy agents" to "agents are in production" from months to weeks.

## Essential Resources (3)

1. **GitHub** — [How Squad Runs Coordinated AI Agents Inside Your Repository](https://github.blog/ai-and-ml/github-copilot/how-squad-runs-coordinated-ai-agents-inside-your-repository/) — Document that multi-agent coordination through repository state (commits, branches, PRs) is now a shipped product, not a future vision—directly applicable if you're building parallel agent workflows.

2. **LangChain** — [Anatomy of an Agent Harness + Autonomous Context Compression](https://blog.langchain.com/the-anatomy-of-an-agent-harness/) — Framework arguing that Agent = Model + Harness, and that harness engineering (tool integration, memory, error recovery, context management) is where differentiation lives—useful mental model for diagnosing why agent quality degrades and what to fix first.

3. **Cursor + Notion** — [Self-Hosted Cloud Agents](https://cursor.com/changelog/03-25-26) and [Custom Agents with Security Architecture](https://www.notion.com/blog/introducing-custom-agents) — Together show that agents are graduating from developer tools (Cursor) to mainstream productivity platforms (Notion), with enterprise-grade security and data residency as table stakes—clear signal of where the market is heading.
