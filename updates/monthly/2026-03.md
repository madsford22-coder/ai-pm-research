# March 2026 Research Summary

March brought a clear shift in how developer tools think about AI agents: from documentation-first to execution-first. LangChain shipped CLI-based skills packages that let agents use their framework by invoking commands—`langsmith trace add` instead of reading SDK docs—while Vercel added Model Context Protocol support for standardized tool discovery. The message: if agents can't use your product directly without human mediation, you're designing for the wrong user.

Beyond tooling, we're seeing infrastructure providers optimize the unsexy details that compound into velocity gains. Vercel's 15% deployment speedup and decoupled CDN routing don't sound revolutionary, but if you deploy 20 times per day, that friction adds up fast.

## What Matters

- **Agent tooling is converging on executable skills over documentation.** LangChain's skills packages teach agents framework expertise through CLI commands—agents invoke `langsmith trace add` to add tracing rather than reading implementation docs. Vercel added MCP support enabling standardized agent tool discovery across platforms. The architectural shift: agent capabilities expand through composable skills (proprietary) and protocol adoption (open standards), not through bigger context windows or better prompts. For PMs building developer tools, the test becomes: can an agent add your SDK, configure it, and invoke core features without a human reading docs first?

- **Infrastructure velocity comes from removing deployment friction, not just speed.** Vercel shipped 15% faster deployments alongside CDN routing updates that don't require redeployment—decoupling infrastructure configuration from code deployment. Previously, testing different caching strategies meant deploying multiple times; now teams iterate on routing rules in production without code changes. The compound effect: if infrastructure changes make up 20% of your deployments, eliminating those redeployments frees your pipeline for actual code changes. Small operational improvements accumulate into meaningful velocity gains.

- **AI enables new governance patterns for distributed product work.** Teresa's case study on Momental reveals how AI detects "merge conflicts" in product strategy—contradictory OKRs and roadmaps, orphaned initiatives, stale decisions that haven't been archived. The shift: from passive documentation storage to active governance. Just as Git surfaces code conflicts automatically, Momental surfaces strategy misalignment across artifacts. For PMs managing complex product strategies, the question becomes: how much strategy drift exists in your artifacts today, and would automated conflict detection catch real problems or just create noise?
