# March 2026 Research Summary

March revealed the shift from "AI can do this" to "how do we scale this responsibly." Lenny Rachitsky documented nine specialized agents running his actual life while Microsoft and Mistral shipped multi-model systems. By month's end, GitHub's Squad showed agents executing entire workflows autonomously, Notion launched Custom Agents for mainstream teams, and LangChain rebranded as an enterprise platform—signaling agents have moved from founder tools to production infrastructure.

The organizational layer is becoming as important as the capability layer. Teams aren't asking whether agents work anymore; they're asking how to govern them, secure them, debug them at scale, and compose multiple agents into reliable systems. Infrastructure providers are racing to provide the harness around the model—observability (Polly), secure execution (Sandboxes), deployment integration (CLI), and team governance (Fleet)—because that's where differentiation lives when model quality converges.

## What Matters

- **Nine specialized agents beat one general agent.** Lenny Rachitsky documented running nine purpose-built agents across family calendar, inbound sales, kids' homework, and more—proving that users don't want single all-knowing AI, they want orchestrated specialist agents with clear domain boundaries and explicit handoff protocols. This directly contradicts the "build one powerful agent" approach most teams start with. The implication: agent product design should emphasize specialization and composition over generality.

- **Enterprise adoption requires security architecture, not just capability.** Notion shipped Custom Agents with detailed security documentation covering permission scoping, agent isolation, and failure recovery. GitHub published explicit threat models for agentic workflows. Cursor launched self-hosted agents for air-gapped environments. The pattern: enterprise procurement is now gated on documented security architecture, not model quality. Teams that can articulate their security boundaries in writing will close deals faster than those claiming "trust us."

- **Harness engineering is where agent quality actually comes from.** LangChain's framework—Agent = Model + Harness—shows that when model quality converges (all teams access GPT-5.4), differentiation lives in context management, error recovery, tool integration, and output validation. Autonomous context compression, Polly trace analysis, and LangSmith Sandboxes are all harness components being productized. For PMs, this reframes the "which model should we use?" question to "which harness components are we missing?"
