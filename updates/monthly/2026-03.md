# March 2026 Research Summary

March marked the maturation of agent tooling from "getting agents to work" to "making products agent-usable by default." LangChain shipped executable skills packages that teach agents framework expertise through CLI commands rather than documentation, while Vercel optimized deployment pipelines and added Model Context Protocol support. The shift: developer tools competing on whether agents can use them autonomously, not just whether humans can.

The architectural question changed from "can we build agents?" to "can agents build with our tools?" LangChain's answer—CLI interfaces that agents invoke directly—reveals a reference pattern for agent-first developer experience.

## What Matters

- **Agent-first developer tools require executable interfaces, not documentation.** LangChain shipped LangSmith and LangChain skills packages with CLI commands like `langsmith trace add` and `langchain agent build --pattern react`. Agents invoke these commands to add tracing, build test sets, and scaffold agentic workflows—without reading API docs or understanding framework internals. The architectural shift: skills packages abstract complexity through execution rather than explanation. For developer tool builders, the question becomes: can agents use your product autonomously, or do they need human mediation at every step?

- **Infrastructure providers compete on operational velocity, not capability lists.** Vercel shipped 15% faster deployments and decoupled CDN routing from code deployment—enabling teams to modify caching and redirects without rebuilding applications. These aren't feature breakthroughs; they're developer experience refinements that compound over repeated use. Deploy 20 times per day? That 15% speedup returns hours per week. Need to test different caching strategies? Iterate on routing rules without code changes. The pattern: infrastructure differentiation happens in operational details that remove friction from high-frequency workflows.

- **Standards versus ecosystems in agent tooling.** Vercel added Model Context Protocol (MCP) support while LangChain shipped proprietary skills packages—both solving agent capability discovery and composition, but through different architectures. MCP offers open interoperability (any agent can invoke any MCP-compatible tool). LangChain Skills offers controlled ecosystems (faster iteration, tighter integration). The tension: fragmented proprietary formats versus universal protocols. For PMs building agent products, this choice affects integration complexity, ecosystem reach, and how quickly capabilities can compound across tools.
