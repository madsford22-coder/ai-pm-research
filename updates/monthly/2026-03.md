# March 2026 Research Summary

March revealed the shift from "can we build agents?" to "can we operate agents at scale?"—with Stripe proving AI features need rigorous A/B testing (1.5M sessions), Notion shipping Custom Agents as first-class product alongside security architecture, and LangChain rebranding Agent Builder as Fleet for enterprise teams. The month showed agents graduating from developer tools into mainstream productivity, and with that graduation comes the hard infrastructure problems: secure execution, observability, governance, and pricing models that reflect actual business value.

Equally important: the PM role is being redefined by agent velocity. When code generation is automated, product value shifts from detailed specification-writing toward quality judgment and discovery. GitHub's "execution is the new interface" thesis and LangChain's "Agent = Model + Harness" framework both point to the same constraint: the limiting factor is no longer whether agents can produce output, but whether teams can evaluate, govern, and operate that output reliably.

## What Matters

- **Rigor is becoming a product differentiator for AI features.** Stripe co-authored their Adaptive Pricing A/B test (1.5M checkout sessions) with both a PM and data scientist—publishing full methodology, control group design, and statistical validity. This sets an evidentiary bar most AI teams skip entirely. The implication: if you can't write a post like Stripe's (test design, results, confidence intervals), you probably don't know whether your AI feature actually works. For PMs shipping AI with claimed conversion or retention impact, this is the template.

- **Multi-agent coordination and enterprise governance are shipping as products.** GitHub's Squad coordinates multiple agents inside repositories through shared Git state; LangChain rebranded Agent Builder as Fleet with explicit enterprise positioning; Notion shipped Custom Agents with a dedicated security architecture post. The pattern: agents are graduating from "individual developer tools" to "team infrastructure." With that comes pricing restructuring (Windsurf moved from credits to quotas), access controls, audit trails, and management dashboards. For teams running more than a handful of agents, enterprise platforms are no longer optional.

- **Infrastructure for secure agent execution is becoming managed services, not DIY.** Vercel published how Notion delegates untrusted code execution to Vercel Sandbox rather than building in-house; LangChain shipped LangSmith Sandboxes (private preview) for agents to run code safely; GitHub documented explicit security boundaries for agentic workflows. The convergence signal: secure execution environments are infrastructure commodities. Before building custom sandboxing, evaluate whether managed services (Vercel Sandbox, LangSmith Sandboxes) solve your constraints—they often do at enterprise scale.
