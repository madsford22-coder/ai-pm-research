# March 2026 Research Summary

March was quiet but revealing: infrastructure providers competed on operational details instead of capabilities, and agent tooling started converging around a specific pattern—teaching agents through executable commands rather than documentation. LangChain shipped CLI-based skills packages that let agents invoke `langsmith trace add` instead of reading SDK docs, while Vercel decoupled CDN routing from deployments and added 15% faster builds.

The undercurrent: agent-first tooling requires rethinking how we package expertise. Documentation teaches humans concepts. Skills packages give agents commands they can run. That's not just a UX difference—it's an architectural shift in how developer tools get built.

## What Matters

- **Agent tooling converged on executable skills over documentation.** LangChain shipped LangSmith Skills and LangChain Skills—CLI packages that teach agents framework expertise through commands like `langsmith trace add` and `langchain agent build --pattern react`. Agents don't read docs about tracing theory; they invoke commands that add tracing. Vercel added Model Context Protocol support for standardized tool discovery. The pattern: agent capabilities expand through skills composition (proprietary packages) and protocol adoption (open standards), not through better prompts or bigger context windows.

- **Infrastructure velocity became the competitive dimension.** Vercel shipped 15% faster deployments and decoupled CDN routing from code deployment—letting teams tweak caching, redirects, and routing without rebuilding applications. Windsurf added GPT-5.4 the same week OpenAI launched it, with promotional pricing at 1x credits. GitHub announced 60 million Copilot code reviews without quality metrics. The competition: operational polish (speed, model access, pricing) rather than capability breakthroughs.

- **AI-powered governance patterns emerged for cross-functional work.** Teresa documented how Momental uses AI to detect "merge conflicts" in product strategy—when OKRs say "focus on retention" but roadmaps emphasize new user acquisition. The insight: strategy drift happens not because teams are careless, but because different artifacts are owned by different people, decisions get made in meetings but don't update docs, and context lives in heads not writing. AI surfaces these conflicts so teams can align intentionally.
