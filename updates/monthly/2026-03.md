# March 2026 Research Summary

The agent tooling layer is getting its own infrastructure stack. LangChain shipped CLI-based skills packages that teach agents framework expertise through executable commands instead of documentation, while Vercel added Model Context Protocol support for standardized tool discovery—both solving the same problem from different angles: making capabilities composable without requiring agents to "understand" underlying systems.

Meanwhile, infrastructure providers are competing on operational details rather than headline features. Vercel's 15% faster deployments and decoupled CDN routing show the shift toward developer experience optimizations that compound over hundreds of daily operations.

## What Matters

- **Skills packages beat documentation for agent-first tools.** LangChain's approach: instead of teaching agents LangChain concepts through docs, give them CLI commands like `langsmith trace add` and `langchain agent build --pattern react`. Agents don't need to understand tracing implementation—they just invoke the right command. This reveals the architectural pattern for making developer tools agent-usable: executable interfaces with clear commands, not comprehensive API documentation. If your SDK assumes human conceptual understanding, agents can't use it autonomously.

- **Infrastructure velocity shows up in compound gains, not single features.** Vercel's 15% deployment speedup might seem minor—but if you deploy 20 times daily, that's real time saved. Their CDN routing decoupled from deployments eliminates a bottleneck: previously, testing different caching strategies meant triggering full build-test-deploy cycles; now you iterate on routing rules in production without redeploying code. The pattern: infrastructure providers competing on operational details that affect daily workflows, not just capability checklists.

- **Agent capability expansion splits between proprietary and open standards.** LangChain ships proprietary skills packages (CLI tools specific to their ecosystem) while Vercel adds Model Context Protocol support (open standard for agent tool discovery). Both solve composability, but with different architectural bets: proprietary ecosystems enable faster iteration and tighter integration; open protocols enable broader ecosystem growth and reduced lock-in. The tension matters because it determines whether agent tooling fragments across platforms or converges around shared standards.
