# March 2026 Research Summary

Infrastructure providers are competing on developer experience minutiae while agent tooling converges around executable skills. Vercel ships 15% faster deployments and lets you update CDN routing without redeploying code. LangChain packages their entire framework as CLI commands that agents invoke directly—no documentation reading required. The shift: infrastructure details matter more than feature lists, and agent-first tooling means executable interfaces, not comprehensive docs.

For PMs, this month reveals two architectural questions: Can agents use your product without human mediation? And are you competing on operational details (speed, routing flexibility) or just feature parity?

## What Matters

- **Agent tooling is converging around skills, not documentation.** LangChain ships CLI packages that teach agents their framework through commands like `langsmith trace add` and `langchain agent build --pattern react`—agents execute tasks without reading docs. Vercel adds Model Context Protocol support for standardized tool discovery. The pattern: agent capabilities expand through executable interfaces (CLI commands, protocol standards) rather than better prompts or documentation.

- **Infrastructure velocity now includes configuration velocity.** Vercel decouples CDN routing from deployments—you can now modify caching rules, redirects, and traffic routing without rebuilding your application. Previously, tweaking cache headers meant triggering a full build-test-deploy cycle. Now infrastructure iteration happens independently of code changes. The implication: infrastructure teams can experiment with caching strategies without waiting for deployment pipelines.

- **AI enables governance patterns beyond passive storage.** Teresa's case study on Momental shows AI detecting strategy conflicts across product artifacts—flagging when OKRs say "focus on retention" but roadmaps prioritize acquisition features. This extends coordination patterns from February: AI moves from "write faster" to "stay aligned automatically." The question for PMs: how much strategy drift exists in your artifacts, and would automated conflict detection catch real problems or generate noise?

## Essential Resources

1. **LangChain** — [LangSmith CLI & Skills](https://blog.langchain.com/langsmith-cli-skills/) — Reference architecture for making frameworks agent-usable: package domain expertise as executable commands rather than requiring documentation comprehension.

2. **Vercel** — [CDN Routing Without Redeployment](https://vercel.com/changelog/vercels-cdn-now-supports-updating-routing-rules-without-a-new-deployment) — Infrastructure configuration decoupled from code deployment—modify caching and routing rules without triggering builds.

3. **Teresa Torres** — [How Momental Uses AI to Find Strategy Conflicts](https://www.producttalk.org/building-github-for-product-management-how-momental-uses-ai-to-find-merge-conflicts-in-strategy/) — Case study on detecting contradictions across product artifacts (PRDs, OKRs, roadmaps) using AI—shifting from passive documentation to active governance.
