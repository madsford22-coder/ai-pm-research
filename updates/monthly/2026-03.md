# March 2026 Research Summary

March revealed the shift from "agents as experimental feature" to "agents as production infrastructure." Stripe published rigorous A/B test results on AI pricing (1.5M checkout sessions), Notion shipped Custom Agents as a first-class product with security architecture to match, and LangChain rebranded Agent Builder as Fleet—explicitly positioning for enterprise-wide agent deployment. The month documents agents graduating from prototype to platform.

The secondary move: infrastructure converging on secure execution and operational governance. Vercel published how Notion runs untrusted code at scale via Sandbox, GitHub documented security architecture for agentic workflows, and LangChain shipped autonomous context compression alongside a "harness framework" explaining where agent quality actually comes from (it's not the model—it's the execution environment surrounding it). For PMs, this means the hard problems are no longer "can we build agents?" but "can we operate them safely and at scale?"

## What Matters

- **Agents are becoming team infrastructure, not individual tools.** LangChain rebranded Agent Builder as Fleet with explicit enterprise positioning ("a central place for all of your teams"), while Notion shipped Custom Agents with four companion posts (announcement, security architecture, internal usage, customer case study)—the launch pattern of a mature feature, not an experiment. Windsurf restructured pricing from credits to quotas with a new Max plan for power users. The signal: individual productivity tools are graduating to team platforms with pricing, security, and governance designed for org-wide adoption.

- **Secure code execution and operational observability are now table stakes for production agents.** Vercel published how Notion delegates sandbox execution to managed infrastructure rather than building it internally, GitHub documented explicit security boundaries for agentic workflows (agent-to-sandbox, agent-to-agent, agent-to-model, agent-to-data isolation), and LangChain shipped autonomous context compression to prevent silent failures in long-running agents. Each addresses a specific production failure mode that infrastructure now handles as a managed layer.

- **The PM job is shifting from specification to quality judgment as code generation becomes automated.** LangChain documented that when code is cheap to produce, Product value moves from "explain what to build in detail" to "define what good looks like" and "discover which problems matter." Stripe's A/B test on Adaptive Pricing showed the evidentiary bar rising—"we think this AI feature helps" is no longer enough; you need controlled experiments at scale. The harness framework (Model + Harness, where harness = tool integration, memory, error recovery, context management, guardrails) reveals that model selection matters less than the infrastructure surrounding it.

## Essential Resources (3)

1. **Stripe** — [A/B Testing Adaptive Pricing Across 1.5M Checkout Sessions](https://stripe.com/blog/product) — Rare production rigor: PM and data scientist co-authored methodology post showing how to design, run, and interpret large-scale AI feature experiments with real data instead of vibes.

2. **LangChain** — [The Anatomy of an Agent Harness](https://blog.langchain.com/the-anatomy-of-an-agent-harness/) — Framework explaining why Agent = Model + Harness; tool integration, memory, context management, error recovery, and guardrails are where agent quality actually comes from, not model selection alone.
