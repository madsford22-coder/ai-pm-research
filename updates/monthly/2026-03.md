# March 2026 Research Summary

March revealed a fundamental shift in how developer tools prepare for agents: instead of writing better documentation, ship executable interfaces. LangChain's CLI-based skills packages taught agents framework expertise through commands like `langsmith trace add`—no SDK knowledge required. Meanwhile, infrastructure providers like Vercel competed on operational details (15% faster deployments, CDN routing without redeployment) rather than flashy capabilities.

The month's signal: agent-first tooling requires rethinking product interfaces entirely. Can an AI use your product without reading docs? If not, you're building for the wrong user.

## What Matters

- **Agent tooling converges on executable skills over documentation.** LangChain shipped CLI packages where agents invoke `langsmith trace analyze` to debug traces instead of parsing SDK documentation. Vercel added Model Context Protocol support for standardized tool discovery across frameworks. The pattern: successful agent tools expose capabilities through composable commands, not comprehensive guides. If your developer product assumes humans read docs first, agents can't use it.

- **Infrastructure velocity becomes the new feature battleground.** Vercel shipped 15% faster deployments and decoupled CDN routing from code deploys—enabling caching changes without rebuilding apps. These aren't headline features; they're operational details that compound daily. For teams deploying 20 times per day, 15% adds up. The competitive shift: infrastructure providers win on developer experience minutiae (deployment speed, config flexibility) rather than capability checklists.

- **AI enables governance patterns previously impossible at scale.** Momental (documented by Teresa Torres) uses LLMs to detect "merge conflicts" in product strategy—flagging when roadmaps contradict OKRs or PRDs drift from goals. The insight: strategy misalignment isn't carelessness; it's multiple people working on related artifacts without coordination. AI surfaces these conflicts automatically, shifting product management from passive documentation to active governance. The question for PMs: how much strategy drift exists in your artifacts, and would automated detection help or create noise?
