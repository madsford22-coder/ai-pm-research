# March 2026 Research Summary

Infrastructure providers are competing on the granular details now—15% faster deployments here, configuration changes without rebuilds there. But the bigger shift is agent tooling converging around a specific pattern: teaching agents through executable interfaces rather than documentation. LangChain shipped CLI-based skills packages that let agents invoke `langsmith trace add` instead of reading SDK docs, while Vercel added Model Context Protocol support for standardized tool discovery.

Meanwhile, Teresa's case study on Momental reveals how AI enables product governance at scale—automatically detecting when your roadmap contradicts your OKRs before anyone notices the mismatch. The March signal: we're past "can AI help?" and into "what operational patterns does AI unlock?"

## What Matters

- **Agent-first tooling means executable interfaces, not better docs.** LangChain's skills packages give agents framework expertise through CLI commands—agents invoke `langsmith trace add` to add tracing rather than reading SDK documentation to implement it manually. This flips the traditional developer tools model: instead of comprehensive docs for humans, ship composable CLI commands that agents execute directly. The architectural implication: if agents can't use your product autonomously today, you're building for the wrong interface layer.

- **Infrastructure details compound into real velocity differences.** Vercel's 15% deployment speedup sounds incremental until you deploy 20 times per day—that's meaningful time saved. Their CDN routing update decouples infrastructure configuration from code deployment, eliminating the "deploy just to change a cache rule" pattern. Windsurf added GPT-5.4 with promotional pricing the same week OpenAI launched it, showing model integration speed as competitive differentiation. The pattern: infrastructure providers competing on operational details (speed, model access, configuration flexibility) rather than headline features.

- **AI enables automated governance patterns for cross-functional work.** Momental uses AI to detect "merge conflicts" in product strategy—when your OKRs say "focus on retention" but your roadmap prioritizes new user acquisition. Traditional PM tools store artifacts; Momental analyzes them for consistency and surfaces contradictions before they derail execution. This extends beyond product management: AI can enforce coordination across any team producing related documents—engineering specs, design mocks, sales decks—by detecting semantic conflicts humans miss.

2. **Teresa Torres** — [How Momental Uses AI to Find Merge Conflicts in Strategy](https://www.producttalk.org/building-github-for-product-management-how-momental-uses-ai-to-find-merge-conflicts-in-strategy/) — Documents AI-powered governance detecting contradictions across product artifacts—OKRs, roadmaps, PRDs—before they cause execution problems.

3. **Vercel** — [CDN Routing Without Redeployment](https://vercel.com/changelog/vercels-cdn-now-supports-updating-routing-rules-without-a-new-deployment) — Decouples infrastructure configuration from code deployment, letting teams iterate on caching and routing rules without triggering builds.
