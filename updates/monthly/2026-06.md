# June 2026 Research Summary

June was the month the AI cost conversation became unavoidable. Uber capped Claude Code usage to manage expenses. Figma introduced pay-as-you-go AI credits instead of flat-rate access. Ravi Mehta coined "tokenmaxxing" to describe the pattern where AI tools consume resources faster than they deliver value. The industry moved from "AI is free once you've built it" to "we need visible dials to control what we're spending."

Underneath the cost reckoning is a deeper shift: the products winning right now aren't the ones with the most capable AI — they're the ones giving enterprises control. Cursor shipped multi-team org management with per-team budgets. LangChain launched RubricMiddleware so agents can self-evaluate before returning outputs. Figma added Plan mode to Figma Make so users can steer AI generation before it happens. The pattern is consistent: visibility and checkpoints before execution.

## What Matters

- **Agent infrastructure is maturing from prototype to production, and the constraint is reliability, not capability.** AWS Bedrock AgentCore, LangChain's isolated sandboxes, and Vercel's Sandbox Drives all shipped isolated execution environments for agents in the same week — a signal that production agent deployment is the real game now, not capability demos. LangChain's RubricMiddleware and DeltaChannel solve two specific production problems: agents need to self-evaluate their work (Harvey published a case study on legal verification), and long-running sessions need storage that doesn't scale O(N²). The Harmonic case study showed 4x retention improvement after rebuilding on Deep Agents — real retention signal, not just capability metrics. The message to PMs is clear: your agent feature won't succeed because the model is good. It will succeed if it's reliable, observable, and doesn't surprise the user.

- **The build vs. buy question for AI is being reframed as a cost control problem.** Uber's Claude Code cap, Figma's pay-as-you-go credits, and Cursor's budget controls per team all point to the same realization: teams need to see and govern token spend the same way they manage any other infrastructure cost. Ravi Mehta's "tokenmaxxing" framework — context pruning, model routing, caching and deduplication — is useful because it's concrete and gives teams a shared language for the cost conversation. But the deeper insight is that the companies winning procurement right now are the ones who give finance and IT real dials to turn. If you're building an AI product for enterprise, budget visibility isn't a nice-to-have — it's table stakes.

- **Memory, forgetting, and scope are the unsolved UX problems, and different companies are betting on different solutions.** OpenAI's "Dreaming" memory system consolidates and prioritizes context instead of just accumulating it — the framing is that forgetting strategically is as important as remembering. Notion explicitly positioned agents as first-class consumers of its Developer Platform, which changes the conversation from "what can my product do" to "what substrate do agents need to work with this?" Perplexity deprecated its Mac app entirely to go all-in on "Personal Computer" — an OS-level agent that controls local files and apps. These aren't feature additions. They're different product identities. The question for any PM right now is: are you designing for the agent as a user, or for the human using an agent? The answer determines your entire architecture.
